| | |
|--------|----------------------------------------------|
| Version | 0.1 |
| Status | Bare Frame |

## Overview

Deploy moto infrastructure services (moto-club, moto-keybox, PostgreSQL) to the k3d cluster as K8s workloads. This is the production-like deployment path — all components run in K8s.

**Scope:**
- `moto-system` namespace for infrastructure
- PostgreSQL for persistent data
- moto-club and moto-keybox as K8s Deployments
- Secrets management for credentials and keys
- RBAC for moto-club
- Makefile targets for deployment

**Out of scope:**
- Remote/cloud cluster deployment
- Helm charts (raw manifests for v1, Helm later)
- Ingress / external access (services accessed via port-forward)
- HA / multi-replica deployment
- Automated certificate management
- GitOps (ArgoCD/Flux)

## Specification

### Architecture

```
k3d cluster "moto"
├── moto-system namespace
│   ├── postgres (StatefulSet)
│   │   ├── moto_club database
│   │   └── moto_keybox database
│   ├── moto-keybox (Deployment)
│   └── moto-club (Deployment)
└── moto-garage-* namespaces (created dynamically by moto-club)
    └── garage pods
```

### Components

#### PostgreSQL

| Property | Value |
|----------|-------|
| Kind | StatefulSet |
| Image | `postgres:16-alpine` |
| Databases | `moto_club`, `moto_keybox` |
| Storage | 1Gi PVC |
| Credentials | Generated password, stored in K8s Secret |
| Readiness | `pg_isready` probe |

Accessible within the cluster at `postgres.moto-system:5432`.

#### moto-keybox

| Property | Value |
|----------|-------|
| Kind | Deployment (1 replica) |
| Image | `localhost:5000/moto-keybox:latest` |
| Ports | 8080 (API), 8081 (health) |
| Secrets mounted | KEK, SVID signing key, service token |
| Database | `moto_keybox` via Secret-sourced URL |
| Health probes | liveness, readiness, startup per [moto-bike.md](moto-bike.md) |

#### moto-club

| Property | Value |
|----------|-------|
| Kind | Deployment (1 replica) |
| Image | `localhost:5000/moto-club:latest` |
| Ports | 8080 (API), 8081 (health), 9090 (metrics) |
| ServiceAccount | `moto-club` with ClusterRole |
| Database | `moto_club` via Secret-sourced URL |
| Keybox URL | `http://moto-keybox.moto-system:8080` |
| Garage image | `localhost:5000/moto-garage:latest` |
| Health probes | liveness, readiness, startup per [moto-bike.md](moto-bike.md) |

### RBAC

moto-club needs a ServiceAccount with a ClusterRole granting cluster-wide access to manage garage namespaces. Required permissions:

| Resources | Verbs |
|-----------|-------|
| namespaces, pods, services, configmaps, secrets, PVCs | full CRUD |
| deployments (apps) | full CRUD |
| networkpolicies (networking.k8s.io) | get, list, create, delete |
| resourcequotas, limitranges | get, list, create, delete |
| tokenreviews (authentication.k8s.io) | create |

### Secrets

All secrets are generated by the `deploy-secrets` target.

| Secret Name | Contents | Used by |
|-------------|----------|---------|
| `postgres-credentials` | Generated DB password | PostgreSQL, connection URLs |
| `keybox-keys` | KEK, SVID signing key, service token | moto-keybox |
| `keybox-db-credentials` | Postgres connection URL for keybox | moto-keybox |
| `club-db-credentials` | Postgres connection URL for club | moto-club |
| `keybox-service-token` | Service token (same as in keybox-keys) | moto-club |

### Container Images

Three images must be built and pushed to the local registry before deployment:

| Image | Nix Package | Description |
|-------|-------------|-------------|
| `moto-garage` | `moto-garage` | Full dev container |
| `moto-club` | `moto-club-image` | Bike base + moto-club binary |
| `moto-keybox` | `moto-keybox-image` | Bike base + moto-keybox-server binary |

Service images use the `mkBike` helper. See [container-system.md](container-system.md).

### Manifest Directory

```
infra/
└── k8s/
    └── moto-system/
        ├── namespace.yaml
        ├── postgres.yaml        # StatefulSet + Service + init ConfigMap
        ├── keybox.yaml          # Deployment + Service
        ├── club.yaml            # Deployment + Service + ServiceAccount + RBAC
        └── kustomization.yaml
```

### Ordering Dependencies

```
postgres
  ↓
moto-keybox (needs postgres)
  ↓
moto-club (needs postgres + keybox)
  ↓
garage pods (created by moto-club)
```

K8s handles ordering via readiness probes — each service won't become ready until its dependencies are healthy.

### Deployment Flow

```bash
# 1. Cluster must exist
moto cluster init

# 2. Build and push all images
make build-garage push-garage
make build-club push-club
make build-keybox push-keybox

# 3. Generate and apply secrets
make deploy-secrets

# 4. Deploy infrastructure
make deploy-system

# 5. Port-forward for CLI access
kubectl -n moto-system port-forward svc/moto-club 8080:8080 &

# 6. Use it
MOTO_USER=nick moto garage open
```

### CLI Access

The CLI needs to reach moto-club. Two options:

| Option | Pros | Cons |
|--------|------|------|
| Port-forward | Simple, no config changes | Background process, can drop |
| NodePort service | Stable, no background process | Requires additional Service resource |

### Makefile Targets

| Target | Description |
|--------|-------------|
| `build-club` | Build moto-club container image |
| `push-club` | Push moto-club to local registry |
| `build-keybox` | Build moto-keybox container image |
| `push-keybox` | Push moto-keybox to local registry |
| `deploy-secrets` | Generate and apply K8s secrets to moto-system |
| `deploy-system` | Deploy all moto-system components |
| `deploy-status` | Show status of moto-system pods |
| `undeploy-system` | Delete moto-system namespace |

### Troubleshooting

| Problem | Solution |
|---------|----------|
| Image pull fails | Verify registry: `curl localhost:5000/v2/_catalog` |
| Postgres pod pending | Check PVC binding |
| moto-club CrashLoopBackOff | Check logs — likely missing DB or keybox |
| RBAC permission denied | Check ClusterRoleBinding exists |

## Deferred

- Helm chart packaging
- Ingress controller for external access
- TLS termination
- Multi-replica deployments
- Automated database backups
- Prometheus / Grafana monitoring stack
- moto-club-db migration Job (auto-run migrations before club starts)
